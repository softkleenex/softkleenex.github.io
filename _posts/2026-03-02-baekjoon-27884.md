풀이 

어떻게 해야할까 싶어,, https://velog.io/@ttc1018/%EB%B0%B1%EC%A4%80-27884
이 블로그를 참조하여 n의 크기가 작기때문에 완전탐색으로 풀린다는것을 확인했다.

그 이후에는 mod를 하는것을 깜빡하여 실패, nowans를 갱신할떄 MOD하는것으로 바꾸었다.
그 이후에는 이런저런 이유를 계속 찾다가, 여러번 실패했는데, nowans말고도 ans때에도 MOD하여 성공하였다.

n의 크기를 보고도 풀이를 떠올리는 것과, 자잘한 오류를 직접 찾아내는 힘이 필요한 것 같아.




---
```python

# https://www.acmicpc.net/problem/27884

import sys
import collections
import itertools

def check_m(now):
    maxans = 1
    ans = 1
    for i in range(0, len(now) - 1):
        if now[i] != now[i + 1]:
            ans += 1
            maxans = ans if ans >= maxans else maxans
        else:
            ans = 1
    
    return maxans
            
n, m = map(int, input().split())
MOD = 10**9 + 7
ans = 0


for p in itertools.product([0, 1], repeat = n):
    if check_m(p) == m:       
        nowans = 1
        for v2 in p:
            if v2 == 0:
                nowans = (nowans * 5) % MOD
            else:
                nowans = (nowans * 11) % MOD
        ans = (ans+ nowans) % MOD
        nowans = 1
        

print(ans)

```
